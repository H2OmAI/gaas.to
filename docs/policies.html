How Policies Work

Evaluation Pipeline

Every agent action passes through the policy evaluation stage:

  1. Scope Matching: Policies check if they apply to this action (based on agent, action type, data sensitivity, jurisdiction)
  2. Condition Checking: Applicable policies evaluate their specific requirements against enriched context
  3. Verdict Assignment: Each policy returns PASS, FAIL, CONDITIONAL, or WARN
  4. Conflict Resolution: If multiple policies fire, the most restrictive verdict wins
  5. Risk Scoring: Policies contribute to a 6-dimensional risk score (privacy, security, compliance, ethical, operational, financial)

Verdict Types

  • PASS: Policy requirements are satisfied ‚Äî action may proceed
  • FAIL: Critical policy violation ‚Äî action is immediately blocked (Tier 1 policies only)
  • CONDITIONAL: Policy requires human judgment ‚Äî escalate to deliberation panel
  • WARN: Policy flags a concern but doesn't block ‚Äî recorded in audit trail

Configuration Options

Tier 3 custom policies support organization-specific configuration:

  • Thresholds: Adjust risk tolerance, approval limits, performance minimums
  • Routing Rules: Define which reviewers handle which types of escalations
  • Approval Workflows: Configure multi-step approval chains
  • Namespace Mappings: Define data isolation boundaries
